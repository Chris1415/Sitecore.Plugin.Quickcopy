# ADR-0001: Use ADRs as the architecture backbone

## Status

Accepted

## Context

The product needs a durable, reviewable record of significant technical decisions without burying them only in chat or a single architecture document.

## Decision

Record major architectural choices as Architecture Decision Records (ADRs) in this folder, one primary decision per ADR when practical, and keep this README index up to date.

## Consequences

- Decisions are traceable and versioned with the product tree.
- Planning and implementation commands can reference ADR ids in PRDs and task breakdowns.
