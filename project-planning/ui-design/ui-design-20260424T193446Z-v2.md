## 1. Design Direction

**Single-sentence principle.** The panel is a designer's control deck: an asymmetric, geometric composition where four square action cards orbit a wide Share Link "submit bar," lit by a soft diagonal Blok gradient mesh and grounded by typographic glyphs instead of pictographic icons.

**Why this direction.**
- v1 ("Compact Utilitarian") earned its keep with restraint. v2 must contrast — distinct enough that the user-test panel can compare *direction*, not just spacing tweaks.
- The previous v2 ("stacked buttons") was rejected as not bold enough. Spotlight Compass commits to a real spatial idea: a 2×2 + 1 composition with atmospheric depth and a dominant CTA.
- Personality comes from **layout geometry, typography, and atmospheric depth** — not from custom colour or custom type. **Every fill, stroke, and font is Blok-shipped.** The mesh gradient uses Blok `--primary` + `--accent` at low alpha, so even the atmosphere is token-derived.
- Glyphs are typographic (`↗ ◉ # Aa ↑`) rendered in **Geist Mono** rather than pictographic Lucide icons. This reads as a deliberate design choice (control-panel / typesetting), not as "we picked icons from a library." Blok ships Geist Mono — no extra font budget.

**What we are explicitly NOT doing.**
- No generic AI dashboard aesthetic (no glass-morphism, no neon underglow, no Inter, no Space Grotesk, no Manrope, no JetBrains Mono, no rounded-everything).
- No bespoke palette: every fill/stroke maps to a Blok semantic token. No raw hex. No bespoke HSL outside the `:root` / `.dark` token definitions themselves.
- No animation budget beyond the 280ms copied morph and a 120ms hover lift — this is a tool, not a toy.

## 2. Layout & Structure

**Panel:** 320px wide, fits without vertical scroll, no horizontal scroll. Composition is **asymmetric and grid-anchored**:

```
┌────────────────────────────────────────────┐  ← 320px
│  QUICKCOPY                       [ Dark ⏾ ] │  ← slim header (32px)
│  Spring Campaign Landing Page              │
│  · Northwind Marketing                     │  ← kicker (page title · site)
│  ──────────────────────────────────────────│
│  ╔══════════════╦══════════════╗           │
│  ║  [L]         ║  [P]         ║           │
│  ║              ║              ║           │
│  ║      ↗       ║      ◉       ║           │
│  ║   Live URL   ║  Preview URL ║           │
│  ╠══════════════╬══════════════╣           │  ← 2×2 grid of square cards
│  ║  [I]         ║  [T]         ║           │     (atop diagonal gradient mesh)
│  ║              ║              ║           │
│  ║      #       ║      Aa      ║           │
│  ║   Item ID    ║  Page Title  ║           │
│  ╚══════════════╩══════════════╝           │
│   ┌────────────────────────────────────┐   │  ← Share Link strip (inset)
│   │  ↑   Share Link            [ S ] ⌄ │   │     primary CTA, full strip
│   └────────────────────────────────────┘   │
│  Alt+L · Alt+P · Alt+I · Alt+T · Alt+S     │  ← legend strip
└────────────────────────────────────────────┘
```

**Spatial rules.**

| Slot | Blok spacing | Approx px | Rationale |
|------|-------------|-----------|-----------|
| Outer panel padding | `p-4` | 16px | Standard Blok panel inset |
| Header → kicker | `mt-1` | 4px | Tight pairing |
| Kicker → grid | `mt-3` | 12px | Breathing room before cards |
| Grid cell gap | `gap-px` then `gap-2` | 1px hairlines + 8px inner pad | Hairline gridlines via `bg-border` |
| Grid → Share strip | `mt-3` | 12px | Visual demotion of grid, promotion of strip |
| Strip inset (left/right) | `mx-2` | 8px each side | Strip is **slightly inset** so the grid extends slightly past it — anchors the composition |
| Strip → legend | `mt-3` | 12px | Final breathing room |

**Vertical budget (verified).** Header 32 + kicker 32 + grid 244 (2×120 + 4 padding) + strip 44 + legend 24 + paddings ≈ 432px. Fits a typical 480-560px Marketplace right-rail without scroll.

## 3. Components

Every interactive element composes from Blok primitives (ADR-0003). The geometry is achieved with Blok layout/spacing tokens — not custom components.

| Component | Blok primitive | Variant / tokens | Role |
|-----------|---------------|------------------|------|
| Action card (×4) | `@blok/button` (rendered as full-area click target) | `variant="ghost"` base + idle border `border-border`, hover `border-primary/40` + `bg-primary/8`, focus `ring-ring` | Live, Preview, Item, Title cards |
| Card glyph | Plain `<span>`, **Geist Mono** | 32px, `text-primary`, font-weight 600 | Typographic glyph (↗ ◉ # Aa) |
| Card label | Plain `<span>`, **Geist Sans** | `text-xs` (12px), `text-foreground/80`, font medium | "Live URL" etc. |
| Card shortcut chip | `@blok/kbd` | Default (mono by Blok convention) | `L` `P` `I` `T` in top-left |
| Share Link primary zone | `@blok/button` | `variant="default"` (primary), `size="sm"`, `flex-1`, rounded-left only | Markdown copy (default) |
| Share Link caret zone | `@blok/button` | `variant="default"`, `size="sm"`, `w-8`, rounded-right only, `aria-haspopup="menu"` | Opens format dropdown |
| Share Link dropdown | `@blok/dropdown` | `align="end"`, `sideOffset={4}` | "Copy as Markdown" / "Copy as Plain text" |
| Theme toggle | `@blok/button` | `variant="ghost"`, `size="xs"`, pill shape | Header right; "Dark ⏾" / "Light ☀" |
| Tooltip (disabled / error) | `@blok/tooltip` | Default | Disabled reason / error message |
| Legend chips | `@blok/kbd` | Default, `text-[10px]` | Bottom strip |
| Gradient mesh | Pure CSS (`radial-gradient`) — `hsl(var(--primary) / x)` + `hsl(var(--accent) / x)` only | Behind grid only | Atmospheric depth |
| Grain overlay | Inline SVG `<feTurbulence>` data-URI | `opacity-[0.022]` light / `0.03` dark, `pointer-events-none` | Optical texture |

**No `Card` wrapper around the panel itself** — the host iframe is the boundary. The 2×2 grid uses CSS grid with hairline `bg-border` gaps to draw the gridlines.

## 4. Color & Typography

### 4a. Colour tokens — **Blok semantic tokens only, no raw hex**

The prototype defines these CSS custom properties at `:root` (light) and `.dark` (dark). Values are reasonable Blok-ish HSL pairs, declared **once** in the token block and never reused as raw HSL elsewhere — every other rule references them via `hsl(var(--token))` or `hsl(var(--token) / <alpha>)`.

| Slot | Token reference |
|------|-----------------|
| Panel surface / body text | `hsl(var(--background))` / `hsl(var(--foreground))` — both flip via `.dark` |
| Mesh — TL / mid / BR lobes | `--primary @ 0.10/0.18`, `--accent @ 0.08/0.12`, `--primary @ 0.06/0.10` (light/dark) |
| Grain | `hsl(var(--grain-color))` at 0.022 light / 0.03 dark via SVG `feTurbulence` |
| Card idle / hover / focus | border `--border`; hover wash `--primary/0.08` + inset ring `--primary/0.4`; focus `--ring` |
| Card glyph / label | `--primary` (Geist Mono) / `--foreground/0.82` (Geist Sans) |
| Share Link bg / fg / hover wash | `--primary` / `--primary-foreground` / `--primary-foreground/0.08` |
| Copied state | bg `--primary/0.14` + inset ring `--primary/0.5`. Success is conveyed by morph + `✓` glyph, not by a separate colour. |
| Error ring / glyph / label | all `--destructive` (ADR-0009 persistent) |
| Disabled hatch | `repeating-linear-gradient(45deg, transparent 0 6px, hsl(var(--muted-foreground) / 0.18) 6px 7px)` |
| Kbd chip / separator | `--muted` + `--muted-foreground` + `--border` stroke / `--border` hairline |

The full Blok semantic surface used: `--primary`, `--primary-foreground`, `--background`, `--foreground`, `--muted`, `--muted-foreground`, `--accent`, `--accent-foreground`, `--destructive`, `--destructive-foreground`, `--border`, `--ring`. Nothing else.

### 4b. Type — **Geist Sans + Geist Mono only (Blok Nova preset)**

Blok ships **Geist Sans** (`--font-sans`) and **Geist Mono** (`--font-mono`) via the Nova preset. No other faces are permitted. The prototype loads them from Google Fonts:

```html
<link href="https://fonts.googleapis.com/css2?family=Geist:wght@300..900&family=Geist+Mono:wght@400..700&display=swap" rel="stylesheet">
```

**Production mapping note.** The prototype's Google-Fonts Geist maps to scaffolded `next/font/google` Geist per Blok `setup.md` — `--font-sans` and `--font-mono` are wired to the same faces, so token swap is identity.

| Slot | Family | Size | Weight | Tracking |
|------|--------|------|--------|----------|
| Wordmark "QUICKCOPY" | Geist Mono | 11px | 500 | `tracking-[0.18em]`, uppercase |
| Kicker (page title) | Geist Sans | 13px | 600 | `tracking-tight` |
| Kicker (site, after `·`) | Geist Sans | 12px | 400 | `text-foreground/70` |
| Card glyph (↗ ◉ # Aa) | **Geist Mono** | 32px | 600 | normal |
| Card label | Geist Sans | 12px | 500 | `tracking-tight` |
| Share Link label | Geist Sans | 13px | 600 | normal |
| Share Link glyph (↑) | **Geist Mono** | 16px | 500 | normal |
| Kbd chips | **Geist Mono** | 10px | 500 | normal |
| Legend captions | Geist Sans | 9px | 500 | `tracking-wide`, uppercase |
| Demo-status / page caption | **Geist Mono** | 11px | 500 | normal — these are value previews / environmental labels |

**Mono rule.** Geist Mono is reserved for: typographic glyphs, `kbd` chips, value previews, and environmental labels (status / caption / wordmark). Everything else is Geist Sans.

### 4c. Contrast (eyeballed AA)

- Card glyph (`primary` on `background`): Blok primary against Blok surface clears 4.5:1 (Blok ships AA-compliant defaults — verified by ADR-0003).
- Share Link (`primary-foreground` on `primary`): Blok-paired tokens — AA by construction.
- Kicker site name at 70% opacity on `background`: above 4.5:1 in both themes.
- Legend chips (`muted-foreground` on `muted`): Blok default — AA.

## 5. Interaction States

### 5a. Idle
- Cards: hairline `border-border`, neutral background (mesh shows through). Glyph in `text-primary` (Geist Mono). Label in `text-foreground/80` (Geist Sans).
- Share Link: solid `bg-primary`, `text-primary-foreground`. Caret zone separated by 1px `bg-primary-foreground/25` divider.

### 5b. Hover (cards)
- Inset 1px ring at `primary/40`, background tints to `primary/8`. Glyph nudges +0.5px on Y over 120ms `cubic-bezier(0.2, 0.8, 0.2, 1)`. No scale.
- Cursor: `pointer`.

### 5c. Focus
- `outline: 2px solid hsl(var(--ring))` with `outline-offset: -2px` on the card or strip. Tab order: theme toggle → Live → Preview → Item → Title → Share Link primary → Share Link caret.

### 5d. Active / pressed
- Cards: brief inward press — `scale(0.98)` over 80ms.
- Share Link: Blok default `:active` darken via primary-foreground wash.

### 5e. Copied (the morph)
- **Duration:** 280ms `cubic-bezier(0.34, 1.56, 0.64, 1)` (slight overshoot — feels designed, not bouncy).
- **Card morph:** rotate `4deg` + `scale(1.05)` peak at 140ms, settles to `0deg / scale(1)` at 280ms. Glyph swaps to `✓`. Background tints `primary/14`, inset ring lifts to `primary/50`. Label swaps to `Copied`.
- **Share Link morph:** scale `1.02` peak (no rotation — the strip is wide, rotation reads wrong). Label flips `Share Link` → `Copied`. Strip retains `bg-primary`; copied affordance is conveyed by inset ring `primary-foreground/40` + glyph swap.
- **Hold:** 1500ms (FR-009). No toast.
- **Revert:** label and glyph cross-fade back over 120ms.
- **Re-trigger:** clicking again during the 1500ms restarts the timer (LRU cache key already matches per ADR-0007 — no second fetch).

### 5f. Disabled (Live URL on unpublished page)
- Diagonal hatched overlay across the card via `repeating-linear-gradient` of `muted-foreground/0.18`.
- Glyph at `opacity-50`, label at `opacity-60`.
- `aria-disabled="true"`, `cursor-not-allowed`.
- Tooltip on hover/focus: **"Not published to Edge yet — publish the page first."** (FR-013, ADR-0001 § alignment.)

### 5g. Error (ADR-0009 persistent)
- Affected card: `box-shadow: inset 0 0 0 2px hsl(var(--destructive))` permanently.
- Glyph swaps to `❌` in `text-destructive`.
- Label swaps to `Failed`.
- Tooltip carries the error reason (e.g., `"Couldn't read item ID — try again or refresh the panel."`).
- **No auto-retry.** Persists until cache key changes (page navigation, re-mount, or fresh successful fetch resets it).

### 5h. Theme toggle
- Pill with current mode label + glyph. Click flips `<html class="dark">`. Persists to `localStorage[quickcopy.poc.theme.v2]`. Crossfade body bg/fg over 180ms.

### 5i. Keyboard (ADR-0008)
- `Alt+L` Live · `Alt+P` Preview · `Alt+I` Item · `Alt+T` Title · `Alt+S` Share Link Markdown (default; does **not** open dropdown).
- Iframe-scoped via window-level keydown.
- Suppressed when `event.target.matches('input, textarea, select, [contenteditable="true"]')` or active element is editable.
- Triggers the same morph as a click on the corresponding control.

### 5j. Share Link dropdown
- Caret zone opens dropdown only on click/Enter/Space — never auto-opens.
- Items: "Copy as Markdown" (default, checked indicator), "Copy as Plain text".
- `Esc` closes; arrow keys navigate; `Enter` activates.

## 6. Accessibility

| Requirement | Approach |
|-------------|----------|
| **WCAG AA contrast** | Blok-paired tokens for all foreground/background combos; verified eyeball pass for both themes (§ 4c). |
| **Focus visible** | `outline: 2px solid hsl(var(--ring))` on every interactive element. No `outline: none` without replacement. |
| **Keyboard equivalence** | Every action reachable via Tab; `Alt+` shortcuts mirror click (ADR-0008). Dropdown follows ARIA menu pattern via Blok primitive. |
| **Editable-element suppression** | Shortcut handler bails when `document.activeElement` is `INPUT`, `TEXTAREA`, `SELECT`, or `[contenteditable]` — preserves host editor typing. |
| **Screen-reader names** | Each card: `<button aria-label="Copy Live URL — shortcut Alt+L">`. Glyphs marked `aria-hidden="true"`. Status announced via `aria-live="polite"` region: `"Live URL copied"` / `"Live URL failed: <reason>"`. |
| **Reduced motion** | `@media (prefers-reduced-motion: reduce)` strips rotation + scale; the morph becomes a colour + label cross-fade only. |
| **Disabled state semantics** | `aria-disabled="true"` + tooltip; not `disabled` attribute (so it remains focusable for SR users to hear the reason). |
| **Error state semantics** | `role="alert"` on the persistent error tooltip when first surfaced; tooltip remains available on hover/focus thereafter. |
| **Touch target** | Cards are ~120×120px — well above 44×44 AA minimum. Share Link strip is ~36px tall × full width — meets target. |
| **Theme toggle label** | Visible text "Dark"/"Light" + glyph; `aria-pressed` reflects state. |
