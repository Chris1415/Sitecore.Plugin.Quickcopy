# UI Design — QuickCopy v1: Page Card Hub

**Variant:** v1 (editorial, refined)
**Direction:** "Page Card Hub" — the QuickCopy panel reads as a small magazine-cover card representing the page being edited. The page title is the typographic hero; quick actions become elegant circular icon chips; the share action is a hero CTA pinned at the bottom.
**Replaces:** prior v1 ("stacked button list") — explicitly rejected.

---

## 1. Design Direction

QuickCopy v1 reframes the panel from "list of buttons" to **editorial card**. The user is editing *one specific page*, so the panel celebrates that page: large display-serif title, a single refined metadata caption (site · language), four circular icon chips for quick copies, and one full-width Share Link CTA at the bottom.

Design tenets:
- **Page is the hero.** The title dominates roughly 35% of vertical space, set in display serif at editorial scale.
- **Atmosphere over chrome.** Subtle ambient gradient (diagonal soft glow from top-left), hairline dividers, no panels-within-panels.
- **Concrete previews on hover.** The "killer" detail: hovering or focusing an icon chip surfaces a floating preview card showing the *exact value* that will be copied (truncated). Removes the abstractness of icon-only buttons.
- **Share is the hero action.** Sharing a page link is the most common workflow signal QuickCopy carries; it earns the bottom-pinned full-width CTA.
- **Blok-native, but expressive.** All components, color tokens, and primary action color stay Blok-native. Distinction comes from typography, spatial composition, and motion — not from custom chrome.

Anti-patterns explicitly avoided: vertically stacked button lists, generic dashboard card grids, kbd chips next to every label, native `title` tooltips, toast notifications.

---

## 2. Layout & Structure

Single-column 320px panel. Vertical regions, top → bottom:

| Region | Height | Purpose |
|---|---|---|
| **Slim header** | 36px | Wordmark `QuickCopy` (small, tracked) on the left; theme toggle (icon-only) on the right. Hairline divider underneath. |
| **Kicker label** | 20px | Small uppercase tracked label `EDITING →`, left-aligned, low-contrast. |
| **Page title (hero)** | ~35% | Display serif, two lines max, balanced wrap, generous line-height. |
| **Metadata caption** | 20px | Single line: `Northwind Marketing · en` in muted caption style. Doubles as the "Copied" announcement slot during the morph. |
| **Icon chip row** | ~80px | Four 36px circular chips, evenly spaced, centered horizontally. Captioned hover preview card floats above on hover/focus. |
| **Spacer** | flex | Pushes share CTA to bottom; preserves vertical asymmetry. |
| **Share Link CTA** | 44px | Full-width split-button, primary color, leading share icon, label `Share this page`, caret zone on the right. |
| **Shortcut legend** | 20px | Single line of small Kbd-style chips: `L  P  I  T  S`. Compact. |

No horizontal scroll. Fits within the typical Pages right-rail height (≥ 560px) without vertical scroll. All padding is symmetric 20px horizontal, with 16px vertical between regions; the title and CTA each get 24px breathing room.

## 3. Components

All components map to Blok primitives (ADR-0003). The variant adds composition and a single new pattern (`HoverPreviewCard`) implemented from Blok primitives — it is not a new design system.

| Component | Blok primitive | v1 customization |
|---|---|---|
| Slim header | `Box` + `Stack` | 36px row, hairline bottom border via `--blok-border-subtle`. |
| Wordmark | `Text` (caption) | Letterspacing `0.14em`, weight 500, uppercase. |
| Theme toggle | `IconButton` (ghost) | Sun / moon glyph; tooltip "Toggle theme". |
| Kicker label | `Text` (overline) | Uppercase, 11px, tracked `0.14em`, color `--blok-text-tertiary`; trailing `→` glyph in serif italic + primary color. |
| Page title | `Heading` (display) | Display serif (see § 4); 28px, 1.12 line-height, weight 500, balanced wrap. |
| Metadata caption | `Text` (caption) | 12px, color `--blok-text-secondary`; renders site · language with a thin middot. |
| Icon chip | `IconButton` (variant: chip) | 36px circle, `--blok-surface-secondary` background, glyph in `--blok-color-primary`. Hover: `translateY(-1px)` + soft shadow. |
| HoverPreviewCard | `Popover` (light) | Floats above the focused chip, 8px gap; rounded 10px; max-width 280px; truncates long values via word-break. |
| Share Link CTA | `SplitButton` (primary) | Full-width; main label `Share this page`; caret opens menu (`Copy as Markdown` default, `Copy as Plain text`). |
| Dropdown menu | Blok `Menu` | Two items only; default item carries `Alt+S` hint chip. |
| Shortcut legend chip | `Kbd` | 20px tall, monospace, single uppercase letter. |
| Demo controls bar (POC only) | `Toolbar` | Stripped to text buttons; not part of the production component. |

**Iconography.** Glyphs are simple stroke icons: globe (Live URL), eye (Preview URL), `#` (Item ID), `aA` (Page Title — typographic glyph, not a doc icon, to keep the editorial register). Share CTA uses an arrow-up-from-tray glyph.

## 4. Color & Typography

### Color (Blok-native)

Light and dark themes use Blok semantic tokens exclusively — no custom palette.

| Surface | Token | Notes |
|---|---|---|
| Panel background | `--blok-surface-canvas` | Receives ambient gradient overlay. |
| Chip / menu surface | `--blok-surface-secondary` / `--blok-surface-elevated` | Subtle elevation, hairline border. |
| Borders | `--blok-border-subtle` / `--blok-border-default` | Hairlines only. |
| Primary action | `--blok-color-primary` | Share CTA fill, chip glyph color, kicker arrow. **Locked: buttons use Blok primary (user explicit).** |
| On-primary text | `--blok-color-on-primary` | Share CTA label. |
| Destructive | `--blok-color-danger` | Error ring + glyph swap on chip. |
| Text primary | `--blok-text-primary` | Page title. |
| Text secondary | `--blok-text-secondary` | Metadata caption. |
| Text tertiary | `--blok-text-tertiary` | Kicker, shortcut legend, disabled glyph. |

**Ambient gradient.** Dark theme adds a soft diagonal glow from top-left using the primary color at low alpha (~10%). Light theme uses the same shape at ~6% alpha. This is decorative; contrast is preserved against the underlying surface token.

### Typography

| Role | Production token | POC value |
|---|---|---|
| Display serif (page title, kicker accent `→`, chip type-glyphs `#` and `aA`) | `--blok-type-display-serif` (mapped at production time; serif display family from Blok kit — small spec follow-up if Blok ships no display serif token today) | **Fraunces** via Google Fonts (`<link rel="stylesheet">` in the HTML), fallback `Georgia, "Times New Roman", serif`. |
| Body / UI | Blok body tokens | `system-ui, -apple-system, "Segoe UI", sans-serif`. |
| Mono (Item ID preview, kbd) | Blok mono token | `ui-monospace, "Cascadia Code", Menlo, monospace`. |

Type scale: Title 28px / 1.12 / 500 with `letter-spacing: -0.01em` and `text-wrap: balance`. Kicker 11px / 1.0 / 500 tracked `0.14em`. Metadata 12px / 1.4 / 400. Shortcut chip 11px / 1.0 / 500. Chip glyph 16px.

## 5. Interaction States

### Default
Chips render filled (secondary surface) with primary-color glyph. Share CTA renders as Blok primary split-button.

### Hover / focus on a chip
1. Chip lifts: `translateY(-1px)`, soft shadow.
2. **HoverPreviewCard** fades in above the chip (8px gap, 120ms ease-out). Card content:
   - Live URL → `https://www.example.com/products/spring-campaign` (mono).
   - Preview URL → preview host equivalent (mono).
   - Item ID → `8E5F4A3B-2C1D-4F9A-9B7E-1A2B3C4D5E6F` (mono).
   - Page Title → `Spring Campaign Landing Page` (sans).
3. Card disappears on blur / mouseleave with 80ms fade.
4. Card is `role="tooltip"` and `aria-hidden="true"`; the value is also exposed via the chip's `aria-label`.

### Click / activate (success morph, 1500ms)
1. Chip scales `1 → 1.15 → 1.0` over 250ms cubic-bezier(0.2, 0.7, 0.2, 1).
2. Glyph swaps to `✓` in primary color.
3. Metadata caption row briefly replaces with `Copied` (semibold, primary-tinted) for 1500ms, then restores `site · language`.
4. Live region (`role="status"`, `aria-live="polite"`) announces e.g. `Live URL copied`.
5. After 1500ms, glyph and caption restore. **No toast.**

For the Share Link CTA: button label flips to `Copied!` with the same scale pulse; caret remains active; restores after 1500ms.

### Disabled (Live URL on unpublished page)
- Chip becomes outline-only (no fill), glyph dimmed to `--blok-text-tertiary`.
- HoverPreviewCard still appears, but content is the explanation: `Not published to Edge yet — publish the page first`.
- `aria-disabled="true"`, focusable, click is a no-op (announces the same message via the live region).
- `Alt+L` is a no-op + announces the same message.

### Persistent error (per ADR-0009)
- Chip gets a `1.5px` ring in `--blok-color-danger` (inset) and the glyph swaps to `✕` in danger color.
- HoverPreviewCard renders the error reason (e.g. `Marketplace SDK timed out — try again`).
- State is **persistent**; clears only on successful retry or when the page-context cache key changes. **No auto-retry.**
- A second click re-attempts; on success the morph runs and the chip returns to default. On failure the error persists.

### Theme toggle
Single icon button, swaps theme + persists to `localStorage["quickcopy.poc.theme.v1"]` in the POC (production: Blok theme provider). Glyph cross-fades 120ms.

### Share Link split-button
- Main click → copy as Markdown (default).
- Caret click → opens menu with `Copy as Markdown` (Alt+S hint) and `Copy as Plain text`.
- `Alt+S` always triggers the default action; never opens the dropdown.

## 6. Accessibility

- **Contrast.** Title and primary action labels meet WCAG AA in both themes. Glyph on chip background and danger ring meet AA at 16px equivalent. The ambient gradient never darkens contrast below the underlying token's compliance.
- **Focus.** All interactive elements receive a visible focus ring (`--blok-focus-ring`, 2px offset). Tab order: theme toggle → chip 1 → 2 → 3 → 4 → Share main → Share caret → demo controls (POC only). Chips are real `<button>` elements; legend chips are not focusable.
- **Keyboard shortcuts (ADR-0008).** `Alt+L/P/I/T/S` invoke the corresponding action. Listener is scoped to the QuickCopy iframe document (not `window.top`). Suppressed when `event.target` is `<input>`, `<textarea>`, `<select>`, or `[contenteditable="true"]`. `Alt+S` triggers Share Link Markdown default and never opens the caret menu.
- **Screen reader.** Each chip has an `aria-label` that names the action (`Copy Live URL`). Live region announces every successful copy and every error. The HoverPreviewCard is decorative (`aria-hidden="true"`) for SR users; the announcement carries the substance.
- **Disabled / error semantics.** `aria-disabled` on the unpublished Live URL chip; `aria-invalid="true"` on the chip in error state.
- **Reduced motion.** When `prefers-reduced-motion: reduce`, the chip scale pulse is replaced with a 250ms opacity dip (1 → 0.6 → 1), the gradient is removed, and HoverPreviewCard fades drop to 40ms.
- **Touch / pointer.** Chips are 36px (above the 32px Blok minimum) with adequate spacing. Acceptable for desktop-only Pages context; if mobile XM Cloud Pages emerges, chips upgrade to 44px.
- **Internationalisation.** Title uses `text-wrap: balance`; non-Latin languages get appropriate font fallback via the system stack. Share menu items are translatable strings, not images.
