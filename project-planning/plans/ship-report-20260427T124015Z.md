## 4. Known Limitations

- **Operator must set `targetHostname` per host record.** Documented in README under "Backend prerequisite â€” `targetHostname`". Without it, `xmc.sites.listHosts` may return placeholder values (e.g. `example.com`) and the Live URL composition will not point at the real public site. QuickCopy reads, never writes, this value.
- **Multi-host fallback rule:** QuickCopy picks the first host with a non-empty `targetHostname`; if none have one, falls back to first non-empty `hostnames[]` entry. Order hosts so the canonical delivery host comes first.
- **Real-tenant smoke is now passing for URL composition with `targetHostname` set.** All other gates from the v0.1 SMOKE_TEST.md remain advisory until a full smoke run is recorded.
