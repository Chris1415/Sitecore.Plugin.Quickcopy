## 1. Summary

The goal of PRD-000 was to ship a Sitecore Marketplace **Page Context Panel** app — five copy buttons (Live URL, Preview URL, Item ID, Page Title, Share Link split-button) plus a theme toggle and a keyboard-shortcut legend — that eliminates the "hunt for the URL" friction marketers hit dozens of times a day inside the Pages editor. The implementation delivered exactly that scope across four sequential phases (Phase 1 scaffold + page-context subscription + cache + clipboard; Phase 2 the four action cards + Share Link split-button; Phase 3 dark-default theme switcher + `Alt+L/P/I/T/S` shortcuts + legend + persistent error model; Phase 4 accessibility polish + jest-axe smoke + bundle measurement + Cloud Portal registration + manual smoke checklist). The cycle ran the full pipeline end-to-end — `/rubber-ducky → /create-prd → /architect → /task-breakdown → /implement → /code-review → /test → /document → /ship` — producing 10 ADRs, a 60-task QA-enriched task breakdown with 150 traceable cases, 24 test files / 172 passing tests, and the complete user-facing documentation set. One critical and one major defect surfaced during code review and were fixed with regression tests; one further major surfaced during test and was also fixed with regression tests. The v0.1 ship gate (T037/T038 manual portal smoke in a real Sitecore tenant) has not yet been executed — the code is complete and verified to the limit of what can be tested outside the Sitecore Pages iframe, hence `shipped_with_caveats`.

## 4. Known Limitations

Things that work as built but carry caveats worth recording for the next iteration.

- **Live URL composition is the documented R-1 approximation.** `xmc.md` does not ship a dedicated `xmc.agent.pagesGetPageLive` SDK key, so live-status detection uses `xmc.pages.retrievePage` and reads `publishing.hasPublishableVersion` + `publishing.isPublishable` (per ADR-0006 / runbook R-1). If Sitecore later ships a dedicated endpoint, the prefetch can be swapped without other code changes.
- **`pageInfo.version` bump on publish is unverified outside Sitecore.** Cache invalidation per ADR-0007 assumes the Pages editor bumps `pageInfo.version` after a publish, which automatically clears any sticky error state. If field testing shows the version does not bump (R-5), the documented v0.2 fallback is a 30-second TTL on the live-status slot.
- **Keyboard shortcuts `Alt+L/P/I/T/S` are provisional.** ADR-0008 commits to the mnemonic scheme but flags conflict-check with the Pages editor as a T038 deliverable. The named fallback `Ctrl+Alt+<letter>` is documented and one constant edit away if a conflict is found.
- **The full panel only renders inside the Sitecore Pages iframe.** Mode A handshake (ADR-0005) requires `window.parent` to broker auth. Outside the portal the panel surfaces its loading state for ~5–10 seconds and then flips to the SDK-init-error state — by design. Local development and Playwright visual smoke can only reach those two surfaces, so the full panel's interaction proof is gated on T038.
- **`role="alert"` on the marketplace SDK init-failure branch carries an implicit `aria-live="assertive"`.** Flagged in the test report as in spiritual tension with ADR-0009 (which forbids `aria-live` for **per-button copy errors**). Two reasonable readings exist: (a) ADR-0009 governs only per-button error spam — a one-time SDK handshake failure is a different error class — or (b) drop to `role="status"` for strict adherence. Not changed in this cycle; surfaced for stakeholder decision.
- **Bundle-size NFR-008 verified via static comment, not a programmatic gate.** Current first-load gz size ~148 KB (~120 KB root-main + ~28 KB panel-specific) inside the 150 KB target. A single new icon library or two heavy Blok primitives could push this over without anything failing red (m-6).

## 6. Technical Debt Introduced

Conscious trade-offs made during implementation, recorded so the next iteration can decide whether to repay them.

- **Clipboard pattern duplicated locally instead of extracted.** `lib/clipboard.ts` is a local copy of pageshot's clipboard pattern (text-only). ADR-0004 commits to the duplication for v0.1 with consolidation deferred to the rule-of-three trigger (a third Marketplace app needing the same pattern).
- **Six minor findings (m-1..m-6) deferred for stakeholder decision.** Listed in Deferred Items § 3; none breach a contract or block ship.
- **Bundle-size NFR-008 verified by static comment, not a programmatic gate (m-6).** Adding a `bundle-size.test.ts` that gzips `.next/static/chunks/*.js` and asserts the budget is straightforward but was deprioritized for v0.1.
- **Live URL composition uses `xmc.pages.retrievePage` + `publishing.{hasPublishableVersion,isPublishable}` as approximation.** Documented in ADR-0006 / runbook R-1. If a dedicated `pagesGetPageLive` ships, the prefetch can swap to it.
- **`/panel` composes directly in `app/panel/page.tsx` rather than a separate `<QuickCopyPanel>` component.** § 10 T036a's planned integration test suite was therefore not built; cross-component flows are partly covered by per-card tests + jest-axe smoke. Surfaced as m-5 / Recommendation 3.
- **`next-themes` dependency was scaffolded but is unused.** QuickCopy uses its own `hooks/useTheme.ts` per ADR-0008. The package remains in `package.json` from the shadcn quickstart scaffold; safe to remove on the next clean-up sweep.
- **Three `as never` casts in `prefetch.ts`** are a Phase-1 workaround for a narrow generic on `client.query` — applied to **request payloads only**, not return values. The regression audit explicitly exempts these three lines; if `as never` ever leaks into a return-value cast, the audit will not catch it (m-3).
